📖 Glosario ▼
Relaciones
Herencia
Una clase "hija" recibe todos los atributos y funciones de la clase "padre".
📌 Ej: Perro es un Animal.
Implementa
Una clase cumple el "contrato" de una interfaz, definiendo sus funciones obligatorias.
📌 Ej: Perro puede Nadar.
Asociación
Una clase usa o conoce a otra, pero ambas existen de forma independiente.
📌 Ej: Alumno usa un Aula.
Composición
Una clase "contiene" a otra. Si el padre se elimina, el hijo también desaparece.
📌 Ej: Casa tiene Habitaciones.
Tipos de clase
C
Clase
Plantilla base para crear objetos. Define atributos (datos) y funciones (comportamiento).
📌 Ej: Perro, Auto, Persona.
A
Abstracta
Clase que NO se puede instanciar directamente. Solo sirve como base para otras clases. Puede tener funciones sin implementar.
📌 Ej: Animal (nadie crea un "Animal", crea un Perro o un Gato).
I
Interfaz
Define un "contrato" de funciones que otras clases deben cumplir. No tiene atributos ni implementación propia.
📌 Ej: Nadador obliga a definir nadar().
Visibilidad
+ public
Cualquier clase puede ver y usar este atributo o función. Visible desde afuera, pero no se hereda automáticamente a los hijos en este diseñador.
📌 Ej: + getNombre(): String
- private
Solo la propia clase puede usarlo. No se hereda ni es visible desde afuera.
📌 Ej: - contraseña: String
# protected
Solo la clase y sus hijos lo reciben. Es la visibilidad correcta para heredar atributos y funciones a clases hijas.
📌 Ej: # calcularImpuesto(): double
